# 04 - 测试结果说明文档

## 一、测试概述
- 测试日期：2026-06-06
- 测试环境：Python 3.9.6，macOS；依赖 flask/requests/pandas/openpyxl 均安装成功
- 测试方式：模块自测 + curl 接口测试 + 实际联网抓取深交所接口

## 二、重要变更（接口修正）

初版误用了「开放式基金净值」报表（`CATALOGID=1953`），该报表对最近交易日存在
发布延迟，导致查询 `159919 / 2026-06-05` 报「该日无数据」。

经排查，深交所「ETF 份额 / 基金规模查询」页面实际调用的是
`CATALOGID=scsj_fund_jjgm` 接口，**必须带 `jjlb=ETF`**，并以 `txtDm` + `txtStart/txtEnd`
查询。改用该接口后问题解决，且能原生按日期区间返回、自带中文简称。
注意：该接口仅提供份额/规模（单位**万份**），不含单位净值，故程序已去除净值字段。

## 五、结论
- 全部测试用例通过；初版报「无数据」的根因（接口选择 + 缺少 `jjlb=ETF`）已修复。

## 六、已知限制与说明
- 数据依赖深交所公开接口，接口结构变化时需同步调整 `szse_client.py`。
- 因深交所证书链不在 Python 默认 CA bundle，请求使用 `verify=False`（已抑制告警）。
- 接口单次查询日期跨度上限约 180 天、每页 20 条，超长区间由程序自动切分并翻页，耗时较长。
- 该接口仅提供份额/规模（万份），不含单位净值。
- 防反爬已加入随机 `random` 参数与窗口/翻页间随机休眠，故回填耗时会相应增加。
- 「跳过已有日期」采用连续覆盖假设（视已有数据覆盖 `[最早,最晚]`）：常规的「先连续回填、之后扩展区间」场景完全正确；若历史数据中间存在因抓取失败造成的空洞，不会自动补抓该空洞（可删除对应 xlsx 后重新回填）。
