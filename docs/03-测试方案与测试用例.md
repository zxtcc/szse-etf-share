# 03 - 测试方案与测试用例文档

## 一、测试目标
验证程序能正确从深交所抓取指定 ETF 指定日期的份额数据、正确入库与去重、
正确进行历史回填与展示，并对异常输入与非交易日给出友好提示。

## 二、测试环境
- Python 3.9+，已安装 `requirements.txt` 依赖
- 可访问 `www.szse.cn`（需联网）
- 启动方式：`python3 app.py`，访问 `http://127.0.0.1:5000`

## 三、测试范围
- 数据抓取模块（szse_client）
- 存储模块（storage）
- Web 接口（/api/query、/api/backfill、/api/history）
- 前端展示（折线图 + 明细表）

## 四、测试用例

| 用例 | 步骤 | 预期结果 |
| --- | --- | --- |
| TC01 正常单日查询 | 代码 159150、日期 2025-09-04 调用 /api/query | ok=true，名称为中文「深证50ETF易方达」，份额约 127,997,623，净值 1.3909 |
| TC02 非交易日查询 | 代码 159150、日期 2025-09-06（周六） | ok=false，提示「该日无数据（可能为非交易日或代码不存在）」 |
| TC03 非法代码 | 代码 `abc`、任意日期 | HTTP 400，提示「请输入正确的 6 位 ETF 代码」 |
| TC04 非法日期 | 代码 159150、日期 `2025/09/04` | HTTP 400，提示日期格式错误 |
| TC05 批量回填 | 代码 159150、区间 2025-09-01~2025-09-05 调用 /api/backfill | ok=true，成功天数=交易日数，跳过天数=周末数，生成 data/159150.xlsx |
| TC06 去重幂等 | 重复执行 TC05 同一区间 | 「文件总记录数」不增加 |
| TC07 历史加载 | 代码 159150 调用 /api/history | ok=true，count>0，data 按日期升序，含份额/净值 |
| TC08 空历史 | 代码 159999（无数据文件） | ok=true，count=0，前端提示先采集数据 |
| TC09 区间过大 | 区间跨度 >400 天 | HTTP 400，提示区间过大 |
| TC10 前端展示 | 浏览器加载历史 | 折线图显示逐日份额趋势，明细表与 xlsx 一致 |

## 五、测试方法
- 模块级：`python3 szse_client.py` 自测抓取；`python3 -c "import storage; ..."` 验证去重。
- 接口级：`curl` 调用三个 API 检查 JSON。
- 界面级：浏览器手动操作查询、回填、加载历史。
