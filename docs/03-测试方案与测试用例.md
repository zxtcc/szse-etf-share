# 03 - 测试方案与测试用例文档

## 一、测试目标
验证程序能正确从深交所抓取指定 ETF 指定日期的份额数据、正确入库与去重、
正确进行历史回填与展示，并对异常输入与非交易日给出友好提示。

## 二、测试环境
- Python 3.9+，已安装 `requirements.txt` 依赖
- 可访问 `www.szse.cn`（需联网）
- 启动方式：在项目根目录执行 `python3 src/app.py`，访问 `http://127.0.0.1:5000`

## 三、测试范围
- 数据抓取模块（szse_client）
- 存储模块（storage：读写、去重、当日变动、汇总、删除、自定义顺序）
- Web 接口（/api/query、/api/backfill[_stream]、/api/history、/api/summary、
  /api/delete、/api/update[_stream]、/api/order）
- 前端展示（折线图、分页明细表、回填/更新进度、汇总表拖拽排序）

## 四、测试用例

| 用例 | 步骤 | 预期结果 |
| --- | --- | --- |
| TC01 正常单日查询 | 代码 159919、日期 2026-06-05 调用 /api/query | ok=true，名称为中文「沪深300ETF嘉实」，份额 1,229,421.66（万份） |
| TC02 非交易日查询 | 代码 159919、日期 2026-06-06（周六） | ok=false，提示「该日无数据（可能为非交易日或代码不存在）」 |
| TC03 非法代码 | 代码 `abc`、任意日期 | HTTP 400，提示「请输入正确的 6 位 ETF 代码」 |
| TC04 非法日期 | 代码 159919、日期 `2026/06/05` | HTTP 400，提示日期格式错误 |
| TC05 批量回填 | 代码 159919、区间 2026-05-01~2026-06-05 调用 /api/backfill | ok=true，抓取交易日数为区间内交易日数（约23），生成 data/159919.xlsx |
| TC06 去重幂等 | 重复执行 TC05 同一区间 | 「文件总记录数」不增加 |
| TC07 历史加载 | 代码 159919 调用 /api/history | ok=true，count>0，data 按日期升序，含份额（万份） |
| TC08 空历史 | 代码 159999（无数据文件） | ok=true，count=0，前端提示先采集数据 |
| TC09 跨多页/多窗口回填 | 区间跨度 >180 天 | 正确切分窗口并翻页，数据完整无缺日 |
| TC10 前端展示 | 浏览器加载历史 | 折线图显示逐日份额趋势，明细表与 xlsx 一致 |
| TC11 跳过已有(全覆盖) | 对已完整存在的区间再回填 | start 事件「待抓取窗口数=0」，不请求深交所，提示已存在 |
| TC12 增量回填 | 在已有数据基础上扩展区间 | 仅抓取缺口区间，已有日期不重复请求 |
| TC13 SSE 进度推送 | 多窗口回填 | 流式返回 start→多条 progress→done，含「当前窗口/总窗口数」 |
| TC14 随机间隔/random | 观察多窗口回填耗时与请求 | 窗口/翻页间有随机休眠，`random` 参数每次不同 |
| TC15 当日变动列 | 查看 xlsx 与 /api/history | 含「当日变动份额(万份)」=当日−前一交易日，首行 0，涨红跌绿 |
| TC16 历史区间过滤 | /api/history 带 start/end | 仅返回区间内记录；区间首行当日变动为相对前一交易日的真实值 |
| TC17 明细表分页 | 加载历史后切页/改每页条数 | 默认每页 20，可选 20/50/100/全部；页码导航正确，表格按日期倒序 |
| TC18 ETF 数据汇总 | /api/summary | 返回各 ETF 代码/名称/数据起止/记录数 |
| TC19 删除 ETF | 勾选后 /api/delete | 删除其 data/代码.xlsx 文件，汇总刷新后消失 |
| TC20 更新数据（含进度） | 勾选后 /api/update_stream | 按默认区间补缺口，流式推送总进度/当前ETF/窗口进度；已最新则 +0 |
| TC21 汇总表自定义排序 | 拖拽行后 /api/order | 顺序持久化到 data/_order.json，刷新后保持，新增 ETF 排末尾 |

## 五、测试方法
- 模块级：`python3 src/szse_client.py` 自测抓取；`cd src && python3 -c "import storage; ..."` 验证去重/当日变动/汇总。
- 接口级：`curl` 调用各 API 检查 JSON；SSE 接口用 `curl -N` 观察流式事件。
- 前端 JS：用 `node --check` 校验内联脚本语法。
- 界面级：浏览器手动操作查询、回填、按区间分析、汇总表删除/更新/拖拽排序。
