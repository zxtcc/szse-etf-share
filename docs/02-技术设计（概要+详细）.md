# 02 - 技术设计文档（概要 + 详细）

## 第一部分：概要设计

### 1. 技术栈
- Web 框架：Flask
- HTTP 抓取：requests
- 数据存储：Excel（pandas + openpyxl），每个 ETF 一个 `data/代码.xlsx`
- 前端图表：Chart.js（CDN 引入），单页 `templates/index.html`

### 2. 系统架构

```
浏览器(index.html, Chart.js)
        |  fetch JSON
        v
   Flask 应用 (app.py)
   /api/query  /api/backfill  /api/history
        |                     |
        v                     v
 szse_client.py          storage.py
 (抓取深交所接口)        (data/代码.xlsx 读写)
        |
        v
  深交所 ShowReport 接口 (CATALOGID=scsj_fund_jjgm, jjlb=ETF)
```

### 3. 模块职责
| 模块 | 职责 |
| --- | --- |
| `szse_client.py` | 封装深交所基金规模接口；`fetch_range` 抓区间逐日份额（切窗+翻页），`fetch_share` 抓单日；非交易日判定。 |
| `storage.py` | `data/代码.xlsx` 的读写、按日期去重、历史序列读取。 |
| `app.py` | Flask 路由与输入校验，串联抓取与存储。 |
| `templates/index.html` | 前端 UI：查询表单、回填、折线图、明细表。 |

## 第二部分：详细设计

### 1. szse_client.py

- `BASE_URL = https://www.szse.cn/api/report/ShowReport/data`，`CATALOG = scsj_fund_jjgm`。
- `_get_json(params, retries=3, timeout=20)`：补全 `SHOWTYPE=JSON`、`TABKEY=tab1`、`CATALOGID`、
  `jjlb=ETF`、随机 `random`；带 `HEADERS`（含 UA、Referer）；`verify=False`；失败重试。
- `_fetch_window(code, start, end)`：抓取 ≤180 天窗口内全部记录，按 `metadata.pagecount` 翻页；
  逐行取 `size_date`/`fund_code`/`security_short_name`/`current_size`（去逗号转 float，单位万份）。
- `fetch_range(code, start, end)`：将大区间按 `MAX_SPAN_DAYS=180` 切分窗口，逐段调用
  `_fetch_window`，按日期去重并升序返回 `{日期, 代码, 名称, 份额}` 列表。
- `fetch_share(code, date)`：`fetch_range(code, date, date)`，取首条或返回 `None`（非交易日/无数据）。

### 2. storage.py

- 列结构：`日期, 代码, 名称, 份额, 抓取时间`（份额单位：万份）。
- `save_records(code, records)`：读旧 `代码.xlsx` → 合并新记录（补 `抓取时间`）→
  `drop_duplicates(subset=["日期"], keep="last")`（**按日期去重，新覆盖旧**）→ 按日期升序 → 写回。返回总记录数。
- `load_history(code)`：读 xlsx → 按日期升序 → 转为 dict 列表（NaN 转 None，份额转 float）。
- `data/` 目录不存在时自动创建。

### 3. app.py 路由
| 路由 | 方法 | 入参 | 处理 |
| --- | --- | --- | --- |
| `/` | GET | - | 渲染 index.html |
| `/api/query` | POST | code, date | 校验 → fetch_share → save_records → 返回单日数据 |
| `/api/backfill` | POST | code, start, end | 校验（跨度≤约6年）→ fetch_range（接口原生区间，自动切窗+翻页）→ save_records → 返回抓取交易日数/文件总数 |
| `/api/history` | GET | code | load_history → 返回逐日序列 |

- 校验：代码 `^\d{6}$`；日期 `^\d{4}-\d{2}-\d{2}$` 且可被 `strptime` 解析。

### 4. 关键算法
- **非交易日判定**：接口对非交易日返回空记录，结果中自然不含该日。
- **窗口切分 + 翻页**：单次查询跨度上限约 180 天、每页 20 条，`fetch_range` 自动按窗口切分并翻页，再按日期去重。
- **去重幂等**：以「日期」为唯一键，重复回填同一区间不会增加行数。
