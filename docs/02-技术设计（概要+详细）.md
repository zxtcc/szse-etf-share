# 02 - 技术设计文档（概要 + 详细）

## 第一部分：概要设计

### 1. 技术栈
- Web 框架：Flask
- HTTP 抓取：requests
- 数据存储：Excel（pandas + openpyxl），每个 ETF 一个 `data/代码.xlsx`
- 前端图表：Chart.js（CDN 引入），单页 `templates/index.html`

### 2. 系统架构

```
浏览器(index.html, Chart.js)
        |  fetch JSON
        v
   Flask 应用 (app.py)
   /api/query  /api/backfill  /api/history
        |                     |
        v                     v
 szse_client.py          storage.py
 (抓取深交所接口)        (data/代码.xlsx 读写)
        |
        v
  深交所 ShowReport 接口 (1953 份额 / 1945 中文名)
```

### 3. 模块职责
| 模块 | 职责 |
| --- | --- |
| `szse_client.py` | 封装深交所接口请求；`fetch_share` 抓单日份额，`fetch_cn_name` 取中文简称；非交易日判定。 |
| `storage.py` | `data/代码.xlsx` 的读写、按日期去重、历史序列读取。 |
| `app.py` | Flask 路由与输入校验，串联抓取与存储。 |
| `templates/index.html` | 前端 UI：查询表单、回填、折线图、明细表。 |

## 第二部分：详细设计

### 1. szse_client.py

- `BASE_URL = https://www.szse.cn/api/report/ShowReport/data`
- `_get_json(params, retries=3, timeout=20)`：补全 `SHOWTYPE=JSON`、`TABKEY=tab1`、随机 `random`；
  带 `HEADERS`（含 UA、Referer）；`verify=False`（深交所证书链不在默认 CA bundle）；失败重试。
- `fetch_share(code, date, with_cn_name=True)`：
  - 请求 `CATALOGID=1953` + `txtQueryDate` + `txtQueryKeyAndJC`。
  - 取 `data[0].metadata.recordcount`；为 0 或 data 空 → 返回 `None`（非交易日/无数据）。
  - 在 data 中按 `fund_code` 精确匹配代码，取 `current_size`（去逗号转 int）、`nav_per_share`（转 float）。
  - 默认补中文名（`fetch_cn_name`），无则退回英文简称。
  - 返回 `{日期, 代码, 名称, 份额, 净值}`。
- `fetch_cn_name(code)`：请求 `CATALOGID=1945`，正则 `<u>(.*?)</u>` 提取 `kzjcurl` 中文简称；进程内缓存。

### 2. storage.py

- 列结构：`日期, 代码, 名称, 份额, 净值, 抓取时间`。
- `save_records(code, records)`：读旧 `代码.xlsx` → 合并新记录（补 `抓取时间`）→
  `drop_duplicates(subset=["日期"], keep="last")`（**按日期去重，新覆盖旧**）→ 按日期升序 → 写回。返回总记录数。
- `load_history(code)`：读 xlsx → 按日期升序 → 转为 dict 列表（NaN 转 None，份额转 int，净值转 float）。
- `data/` 目录不存在时自动创建。

### 3. app.py 路由
| 路由 | 方法 | 入参 | 处理 |
| --- | --- | --- | --- |
| `/` | GET | - | 渲染 index.html |
| `/api/query` | POST | code, date | 校验 → fetch_share → save_records → 返回单日数据 |
| `/api/backfill` | POST | code, start, end | 校验（≤400 天）→ 逐日 fetch_share（None 计跳过，异常计失败，每次 sleep 0.3s）→ save_records → 返回汇总 |
| `/api/history` | GET | code | load_history → 返回逐日序列 |

- 校验：代码 `^\d{6}$`；日期 `^\d{4}-\d{2}-\d{2}$` 且可被 `strptime` 解析。

### 4. 关键算法
- **非交易日判定**：依据接口 `metadata.recordcount == 0`，无需本地交易日历。
- **去重幂等**：以「日期」为唯一键，重复回填同一区间不会增加行数。
