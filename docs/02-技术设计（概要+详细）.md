# 02 - 技术设计文档（概要 + 详细）

## 第一部分：概要设计

### 1. 技术栈
- Web 框架：Flask
- HTTP 抓取：requests
- 数据存储：Excel（pandas + openpyxl），每个 ETF 一个 `data/代码.xlsx`
- 前端图表：Chart.js（CDN 引入），单页 `templates/index.html`

### 2. 系统架构

```
浏览器(index.html, Chart.js)
        |  fetch JSON
        v
   Flask 应用 (app.py)
   /api/query  /api/backfill  /api/history
        |                     |
        v                     v
 szse_client.py          storage.py
 (抓取深交所接口)        (data/代码.xlsx 读写)
        |
        v
  深交所 ShowReport 接口 (CATALOGID=scsj_fund_jjgm, jjlb=ETF)
```

### 3. 模块职责
所有源码位于 `src/` 目录；数据文件存放在项目根目录的 `data/`（与 `src/` 同级）。

| 模块 | 职责 |
| --- | --- |
| `src/szse_client.py` | 封装深交所基金规模接口；`fetch_range` 抓区间逐日份额（切窗+翻页），`fetch_share` 抓单日；非交易日判定。 |
| `src/storage.py` | `data/代码.xlsx` 的读写、按日期去重、历史序列读取。 |
| `src/app.py` | Flask 路由与输入校验，串联抓取与存储。 |
| `src/templates/index.html` | 前端 UI：查询表单、回填、折线图、明细表。 |

## 第二部分：详细设计

### 1. szse_client.py

- `BASE_URL = https://www.szse.cn/api/report/ShowReport/data`，`CATALOG = scsj_fund_jjgm`。
- `_get_json(params, retries=3, timeout=20)`：补全 `SHOWTYPE=JSON`、`TABKEY=tab1`、`CATALOGID`、
  `jjlb=ETF`、随机 `random`；带 `HEADERS`（含 UA、Referer）；`verify=False`；失败重试。
- `fetch_window(code, start, end)`：抓取 ≤180 天窗口内全部记录，按 `metadata.pagecount` 翻页
  （翻页之间 `_sleep(PAGE_SLEEP)` 随机休眠）；逐行取 `size_date`/`fund_code`/
  `security_short_name`/`current_size`（去逗号转 float，单位万份）。
- `split_windows(start, end)`：将大区间按 `MAX_SPAN_DAYS=180` 切分为若干窗口（供回填与进度统计）。
- `_sleep(span)`：在区间内 `random.uniform` 随机休眠，用于反爬。
- `fetch_range(code, start, end)`：`split_windows` 后逐段 `fetch_window`（窗口间 `_sleep(WINDOW_SLEEP)`），
  按日期去重并升序返回 `{日期, 代码, 名称, 份额}` 列表。
- `fetch_share(code, date)`：`fetch_range(code, date, date)`，取首条或返回 `None`（非交易日/无数据）。

### 2. storage.py

- 列结构：内部规范列名 `日期, 代码, 名称, 份额, 抓取时间`（份额单位：万份）；
  写入 Excel 时表头映射为显示标签（`份额` → `份额(万份)`，见 `HEADER_LABELS`），
  `load_df` 读取时用 `REVERSE_LABELS` 还原为规范列名（兼容旧文件）。
- `_write_styled(df, path)`：用 openpyxl 输出**人类友好的 Excel**——工作表名「份额数据」、
  加粗深蓝表头 + 白字居中、冻结首行（`freeze_panes=A2`）、表头筛选器、
  份额列千分位格式（`#,##0.00`）右对齐、按列设置列宽。
- `save_records(code, records)`：读旧 `代码.xlsx` → 合并新记录（补 `抓取时间`）→
  `drop_duplicates(subset=["日期"], keep="last")`（**按日期去重，新覆盖旧**）→ 按日期升序 →
  经 `_write_styled` 写回。返回总记录数。
- `existing_dates(code)`：返回已存储日期的升序去重列表，供回填时跳过已抓取区间。
- `load_history(code)`：读 xlsx → 按日期升序 → 转为 dict 列表（NaN 转 None，份额转 float）。
- `data/` 目录不存在时自动创建。

### 3. app.py 路由
| 路由 | 方法 | 入参 | 处理 |
| --- | --- | --- | --- |
| `/` | GET | - | 渲染 index.html |
| `/api/query` | POST | code, date | 校验 → fetch_share → save_records → 返回单日数据 |
| `/api/backfill_stream` | GET (SSE) | code, start, end | 校验 → `_backfill_events` 流式产出进度，前端 EventSource 实时显示 |
| `/api/backfill` | POST | code, start, end | 与流式同逻辑的非流式版本（消费同一生成器），返回最终汇总 |
| `/api/history` | GET | code | load_history → 返回逐日序列 |

- 校验：代码 `^\d{6}$`；日期 `^\d{4}-\d{2}-\d{2}$` 且可被 `strptime` 解析；跨度≤约 6 年。
- `_backfill_events(code, start, end)`：核心生成器。先 `storage.existing_dates` 读已有日期 →
  `_compute_missing` 算缺口区间 → 切窗口 → 逐窗口 `fetch_window` 抓取并 `yield` 进度事件
  （start / progress / done / error）→ 末尾 `save_records`。
- SSE：`Response(gen(), mimetype="text/event-stream")`，每条事件 `data: {json}\n\n`；
  `app.run(threaded=True)` 以支持流式连接与并发。

### 4. 关键算法
- **跳过已抓取日期（增量回填）**：`_compute_missing` 把已有数据视为覆盖了 `[最早, 最晚]`
  连续区间，仅抓取落在其外的向前/向后增量；区间完全被覆盖时不发任何请求。
- **窗口切分 + 翻页**：单次查询跨度上限约 180 天、每页 20 条，按窗口切分（`split_windows`）并翻页（`fetch_window`）。
- **反爬随机化**：每次请求的 `random` 参数用 `random.random()` 随机生成；
  翻页之间 `PAGE_SLEEP`、窗口之间 `WINDOW_SLEEP` 均为随机区间休眠（`_sleep`）。
- **去重幂等**：以「日期」为唯一键，重复回填同一区间不会增加行数。
