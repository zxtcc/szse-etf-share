# 01 - 需求与流程说明文档

## 一、项目背景与目标

本项目是一个简易的 Web 程序，用于从**深圳证券交易所（深交所）**官网获取指定
ETF（交易型开放式指数基金）在指定日期下的**份额数据**，并在网页上展示；同时
提供该 ETF 历史逐日份额变化的分析功能。

数据来源页面：`http://www.szse.cn/market/fund/volume/etf/index.html`（ETF 份额）。

## 二、功能需求

| 编号 | 功能 | 说明 |
| --- | --- | --- |
| F1 | 单日份额查询 | 用户输入 ETF 代码与日期，点击查询，程序自动从深交所抓取该 ETF 当日份额并展示（名称、代码、日期、份额(万份)）。 |
| F2 | 数据本地保存 | 查询/回填得到的数据保存到 `data/代码.xlsx`（人类友好的 Excel 排版），每个 ETF 一个文件，按日期去重。 |
| F3 | 历史批量回填 | 用户输入代码与起止日期，程序按 180 天分段抓取区间内份额并写入 xlsx；**已抓取过的日期自动跳过**，非交易日自动忽略。 |
| F4 | 历史份额分析 | 读取本地 xlsx，折线图按日期升序展示份额趋势；明细表按日期**倒序**（最近在上）、支持分页（每页 20/50/100/全部，默认 50），并含「当日变动份额(万份)」列（当日−前一交易日，无前序则 0，涨红跌绿）。 |
| F5 | 回填进度展示 | 批量回填过程中，页面通过进度条 + 文字实时显示「第几段 / 共几段、当前时间段、累计抓取天数」。 |
| F6 | ETF 数据汇总 | 独立板块，列出本地已存储数据的所有 ETF（代码、中文名称）及各自数据覆盖的起止日期、记录数；可点击某行跳转到该 ETF 的历史分析。 |

## 三、非功能需求

- 非交易日 / 无数据时给出友好提示，不报错。
- 深交所接口单次最多查 180 天，超出按 180 天分段请求。
- 防反爬：每次请求的 `random` 参数随机生成；分段/翻页请求之间设随机休眠间隔。
- 增量优化：回填前先检查本地已有日期，已有数据的区间不再重复请求深交所。
- 输入校验：ETF 代码为 6 位数字，日期为 `YYYY-MM-DD`。

## 四、用户操作流程

### 流程 A：单日查询
1. 在「单日份额查询」区输入 ETF 代码（如 `159150`）和日期（如 `2025-09-04`）。
2. 点击「查询」。
3. 程序请求深交所接口 → 解析数据 → 存入 `data/159150.xlsx` → 在页面展示结果卡片。
4. 若为非交易日，提示「该日无数据（可能为非交易日或代码不存在）」。

### 流程 B：历史回填与分析
1. 在「历史份额分析」区输入代码、开始日期、结束日期。
2. 点击「批量回填区间」：程序先检查本地已有日期 → 仅对缺口区间按 180 天分段、
   带随机间隔地抓取深交所数据 → 页面进度条实时显示「第几段/共几段、累计天数」→
   完成后写入 xlsx 并自动加载历史。若区间已全部存在，则提示无需请求、直接加载。
3. （或）点击「历史数据分析」，读取已有 xlsx 数据 → 折线图 + 明细表展示逐日份额变化。

## 五、数据来源接口说明

- **基金规模（份额）接口**：`api/report/ShowReport/data`，`CATALOGID=scsj_fund_jjgm`，
  与深交所官网「ETF 份额 / 基金规模查询」页面一致。
  - 必带参数 `jjlb=ETF`（基金类别，缺失会返回「查询异常」）。
  - 查询参数：`txtDm`（ETF 代码）、`txtStart`/`txtEnd`（起止日期，`YYYY-MM-DD`）。
  - 返回字段：`size_date`（日期）、`fund_code`（代码）、`security_short_name`（中文简称）、
    `current_size`（基金规模，单位**万份**）。
  - 接口原生支持日期区间；单次最大跨度约 180 天、每页 20 条，程序自动切窗口 + 翻页。
- 非交易日：该日无记录（`recordcount = 0`），据此判定无数据。
- 说明：该接口只提供份额/规模，不含单位净值。
