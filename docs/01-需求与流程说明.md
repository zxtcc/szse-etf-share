# 01 - 需求与流程说明文档

## 一、项目背景与目标

本项目是一个简易的 Web 程序，用于从**深圳证券交易所（深交所）**官网获取指定
ETF（交易型开放式指数基金）在指定日期下的**份额数据**，并在网页上展示；同时
提供该 ETF 历史逐日份额变化的分析功能。

数据来源页面：`http://www.szse.cn/market/fund/volume/etf/index.html`（ETF 份额）。

## 二、功能需求

| 编号 | 功能 | 说明 |
| --- | --- | --- |
| F1 | 单日份额查询 | 用户输入 ETF 代码与日期，点击查询，程序自动从深交所抓取该 ETF 当日份额并展示（名称、代码、日期、份额、单位净值）。 |
| F2 | 数据本地保存 | 查询/回填得到的数据保存到 `data/代码.xlsx`，每个 ETF 一个文件，按日期去重。 |
| F3 | 历史批量回填 | 用户输入代码与起止日期，程序逐日抓取区间内份额（自动跳过非交易日）并写入 xlsx。 |
| F4 | 历史份额分析 | 读取本地 xlsx，以折线图展示逐日份额变化趋势，并附明细数据表。 |

## 三、非功能需求

- 非交易日 / 无数据时给出友好提示，不报错。
- 对深交所接口请求加入轻微限速与重试，避免请求过快。
- 输入校验：ETF 代码为 6 位数字，日期为 `YYYY-MM-DD`。

## 四、用户操作流程

### 流程 A：单日查询
1. 在「单日份额查询」区输入 ETF 代码（如 `159150`）和日期（如 `2025-09-04`）。
2. 点击「查询」。
3. 程序请求深交所接口 → 解析数据 → 存入 `data/159150.xlsx` → 在页面展示结果卡片。
4. 若为非交易日，提示「该日无数据（可能为非交易日或代码不存在）」。

### 流程 B：历史回填与分析
1. 在「历史份额分析」区输入代码、开始日期、结束日期。
2. 点击「批量回填区间」，程序逐日抓取并入库，完成后返回成功/跳过/失败天数汇总，并自动加载历史。
3. （或）点击「加载历史」，读取已有 xlsx 数据 → 折线图 + 明细表展示逐日份额变化。

## 五、数据来源接口说明

- **份额接口**：`api/report/ShowReport/data`，`CATALOGID=1953`，参数 `txtQueryDate`（日期）、
  `txtQueryKeyAndJC`（代码）。返回字段：`fund_code`、`security_english_short_name`、
  `current_size`（份额，单位「份」）、`nav_per_share`（单位净值）。
- **中文简称接口**：`CATALOGID=1945`（ETF 列表），从 `kzjcurl` 字段 HTML 中提取中文简称。
- 非交易日：接口 `metadata.recordcount = 0`，据此判定无数据。
