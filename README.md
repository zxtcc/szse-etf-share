# 深交所 ETF 份额查询与历史分析 Web 程序

一个简易的 Python Web 程序：从**深圳证券交易所**官网获取指定 ETF 在指定日期的
**份额数据**并在网页展示，同时支持该 ETF 历史逐日份额变化的可视化分析。

数据来源：深交所 ETF 份额页面 `http://www.szse.cn/market/fund/volume/etf/index.html`

## 功能

- **单日查询**：输入 ETF 代码 + 日期，实时抓取深交所份额数据（名称、份额、单位净值）。
- **数据落盘**：查询/回填结果保存到 `data/代码.xlsx`，每个 ETF 一个文件，按日期去重。
- **历史回填**：输入代码与起止日期，逐日抓取区间份额，自动跳过非交易日。
- **历史分析**：折线图展示逐日份额变化趋势 + 明细数据表。

## 启动程序

在**项目根目录**下运行（数据会存到根目录的 `data/`）：

```bash
python3 src/app.py
```

终端出现 `Running on http://127.0.0.1:5000` 即启动成功，浏览器打开：

> http://127.0.0.1:5000

### 修改端口（可选）

默认端口为 5000。**macOS 上 5000 常被系统「AirPlay 接收器」占用**，可用环境变量
`PORT` 指定其它端口启动，无需改代码：

```bash

```

然后访问 http://127.0.0.1:8000 。

## 使用说明

1. **单日份额查询**：填写 ETF 代码（6 位，如 `159150`）和日期（如 `2025-09-04`），点击「查询」。
   非交易日会提示「该日无数据」。
2. **批量回填区间**：填写代码、开始日期、结束日期，点击「批量回填区间」。
   程序按 180 天分段抓取并入库，**页面进度条实时显示抓取进度**；
   已抓取过的日期会自动跳过，分段之间有随机间隔以避免被反爬。
3. **历史分析**：点击「历史数据分析」，以折线图 + 明细表展示该 ETF 的逐日份额变化（读取本地已存数据）。
   明细表支持分页，可选每页 20 / 50 / 100 / 全部，默认每页 50 条。

## 数据文件

- 格式：标准 `.xlsx`（Microsoft Excel），排版对人类友好——加粗表头、冻结首行、
  表头筛选器、份额列千分位显示、合理列宽。
- 路径：`data/代码.xlsx`（如 `data/159919.xlsx`），工作表名「份额数据」。
- 列：`日期, 代码, 名称, 份额(万份), 抓取时间`（**份额单位：万份**，与深交所基金规模口径一致）。
- 同一日期自动去重（重复抓取以最新结果覆盖）。

## 数据来源接口

- 接口：深交所「基金规模查询」`api/report/ShowReport/data`，`CATALOGID=scsj_fund_jjgm`，
  必带 `jjlb=ETF`，参数 `txtDm`（代码）、`txtStart`/`txtEnd`（起止日期）。
- 该接口原生支持日期区间查询，单次最大跨度约 180 天、每页 20 条，
  程序自动按窗口切分并翻页（见 `szse_client.fetch_range`）。
- 返回字段：`size_date`（日期）、`fund_code`（代码）、`security_short_name`（中文简称）、
  `current_size`（基金规模，单位**万份**）。

## 说明与限制

- 数据依赖深交所公开接口，接口结构调整时需同步修改 `szse_client.py`。
- 因深交所证书链不在 Python 默认 CA bundle，请求使用 `verify=False`（已抑制告警）。
- 该接口仅提供份额/规模（万份），不含单位净值。
- 批量回填会按 180 天窗口逐段抓取并翻页，区间很大时耗时较长。
